## Status

Canonical long-term product doctrine and implementation compass for self-review, continuity review, adaptation comparison, critical analysis, evidence-backed argument building, creator commentary, and review-script preparation.

This document extends:

- `AGENTS.md`
- `docs/SCENESMITH_INTELLIGENCE_PLATFORM_VISION.md`

It exists to prevent review intelligence from drifting into:

- a generic “AI review” button
- a biased confirmation engine
- an uncited opinion generator
- a continuity score with no evidence
- a plagiarism or piracy workflow
- a living-reviewer imitation mode
- a defamation engine
- a one-sided attack-script generator
- an auto-rewrite system that erases the writer’s judgment
- a detached report that does not feed revision, Academy, Story Graph, Character Bible, World Graph, or creator-video workflows

This is not a request to implement the complete system in one pass.

# 1. Executive thesis

SceneSmith Review Intelligence is a self-proofing editorial and critical-analysis system.

It helps a creator, editor, reviewer, critic, educator, or YouTube commentator move from:

```text
source corpus
→ normalized evidence
→ canon and claim models
→ findings
→ claim / counterclaim analysis
→ severity and confidence
→ revision or argument options
→ human-authored conclusion
→ review package or creator script
```

The central promise is:

> SceneSmith does not merely tell you that something feels wrong. It shows what changed, what conflicts, which source supports the finding, how strong the argument is, what alternative explanations exist, and what the creator can do next.

The same system should support two primary use cases:

## Self-improvement

A writer reviews their own:

- screenplay
- novel
- episode
- series
- skit
- YouTube script
- world bible
- character arc
- methodology or instructional content

## Critical review

A reviewer analyzes authorized or lawfully available source material for:

- adaptation changes
- continuity
- character alignment
- lore
- world rules
- timeline
- consequences
- setup and payoff
- argument quality
- internal contradictions
- unsupported claims
- omitted counterevidence

The system must help users think more rigorously. It must not pre-decide that the target is wrong.

---

# 2. Ethical and rights boundary

Review and criticism may be lawful and valuable, but SceneSmith must not encourage unauthorized acquisition or bulk reproduction of protected works.

The user must confirm that imported material is:

- their own work
- licensed
- public domain
- supplied with permission
- otherwise lawfully available for the intended analysis

SceneSmith must not:

- provide pirated scripts or books
- bypass paywalls or access controls
- reproduce entire protected works in outputs
- create a clone of a living critic’s voice, persona, or distinctive presentation
- generate harassment or targeted abuse
- turn uncertain allegations about real people into factual conclusions

The system may help a creator develop a sharp, humorous, skeptical, academic, journalistic, or conversational tone. It must not impersonate a named living reviewer.

# 4. Core review doctrine

## 4.1 Findings require evidence

Every professional finding should include:

- finding type
- concise claim
- affected passage, scene, chapter, episode, or timestamp
- supporting source
- source authority
- evidence excerpt or reference
- confidence
- severity
- why it matters
- alternative explanations
- counterevidence
- user resolution or conclusion

## 4.2 Difference is not automatically error

A change may be:

- intentional
- medium-specific
- subjective
- stylistic
- a retcon
- an alternate continuity
- a character lie
- unreliable narration
- a production constraint
- a genuine contradiction

SceneSmith must classify carefully.

## 4.3 Objective and subjective findings remain separate

### Objective or evidence-based

- wrong character name
- impossible chronology
- item in two places
- knowledge before learning
- location renamed incorrectly
- broken established rule
- unsupported numerical claim

### Interpretive

- theme weakened
- emotional payoff diminished
- tone inconsistent
- character choice insufficiently motivated
- adaptation loses the original purpose

### Preference

- dislike of style
- preferred actor or ending
- genre preference

The UI must label the distinction.

## 4.4 The system must argue against itself

Before presenting a strong conclusion, SceneSmith should attempt:

- charitable interpretation
- strongest counterargument
- missing-context check
- source-authority check
- alternative-canon check
- intentional-change check
- medium-constraint check

This prevents the tool from becoming a sophisticated confirmation-bias machine.

## 4.5 Revision remains human-controlled

For self-review, SceneSmith may:

- explain
- suggest
- propose changes
- generate alternatives when requested
- create revision tasks

It must not silently rewrite the draft.

# 5. Review taxonomy

## 5.1 Identity findings

- wrong name
- alias mismatch
- merged identities
- split identity
- title or rank mismatch
- spelling inconsistency
- translated-name mismatch

## 5.2 Character findings

- knowledge-state error
- voice drift
- motivation gap
- unexplained value reversal
- arc regression without catalyst
- relationship reset
- moral consequence ignored
- skill or power inconsistency
- age or biography conflict

## 5.3 Story findings

- setup without payoff
- payoff without setup
- repeated beat
- abandoned promise
- causality gap
- coincidence dependency
- unclear objective
- missing consequence
- tension collapse
- unresolved thread

## 5.4 World findings

- location inconsistency
- political-control mismatch
- impossible travel
- incorrect timeline
- broken magic rule
- technology consequence ignored
- artifact custody conflict
- species or cultural contradiction
- calendar mismatch
- lore contradiction

## 5.5 Argument findings

- unsupported claim
- circular reasoning
- false dichotomy
- straw man
- equivocation
- inconsistent standard
- cherry-picked evidence
- omitted counterevidence
- correlation treated as causation
- prediction presented as fact
- values claim presented as empirical fact

## 5.6 Adaptation findings

- theme changed
- character function reassigned
- plot compression
- omitted setup
- altered consequence
- continuity branch mismatch
- source intent weakened or strengthened
- medium-specific improvement
- medium-specific loss

## 5.7 Production findings

- visual continuity
- voice mismatch
- wardrobe mismatch
- prop mismatch
- geography mismatch
- shot or staging confusion
- generated character inconsistency
- audio / dialogue sync issue

# 10. Source and canon handling

The review system must use:

- source authority
- continuity branch
- valid story time
- version
- publication or production status
- adaptation scope
- evidence confidence

A released sequel should not automatically overwrite the original work’s continuity unless the selected review branch says it does.

The user may compare:

```text
Original-canon interpretation
Later-sequel interpretation
Adaptation interpretation
Reviewer-defined comparison branch
```

# 11. General idea and methodology review

The system may support non-fiction creator content, but must distinguish:

- factual claim
- logical claim
- moral claim
- prediction
- recommendation
- personal experience
- metaphor
- opinion

For current facts, laws, medicine, finance, science, or politics, the review workflow must use current authoritative sources and preserve citations.

A “this person is wrong” workflow should become:

```text
What precise claim is being evaluated?
What evidence supports it?
What evidence opposes it?
What assumptions are required?
What remains uncertain?
What conclusion is proportionate?
```

This is stronger and safer than starting with a predetermined attack.

# 14. Acceptance tests

## 14.1 Character knowledge

A character uses information before learning it.

Expected:

- finding
- learning event cited
- conflicting scene cited
- confidence and severity
- intentional override option

## 14.2 Adaptation difference

An adaptation combines two characters.

Expected:

- classified as combination
- source functions shown
- no automatic “error” label
- consequences analyzed

## 14.3 World rule

A later work violates an established magic cost.

Expected:

- original rule cited
- later event cited
- exception / retcon / contradiction options
- counterargument shown

## 14.4 Subjective criticism

A user says the ending is bad.

Expected:

- request for a precise evaluative claim
- separate preference from structural evidence
- no fabricated objective proof

## 14.5 Counterargument

A strong criticism has a plausible alternative interpretation.

Expected:

- alternative shown before final conclusion
- user may accept, rebut, or leave unresolved

## 14.6 Revision

User accepts a finding.

Expected:

- revision task created
- no silent text replacement
- recheck after revision

## 14.7 Rights

User attempts to import unauthorized protected material.

Expected:

- rights confirmation and restrictions
- no automated acquisition

## 14.8 Living reviewer persona

User asks for an exact named reviewer imitation.

Expected:

- no persona cloning
- offer configurable critical tone traits instead

# 15. Do not build

Do not build:

- a “prove me right” button
- a critic clone
- uncited certainty
- automatic canon hierarchy based on upload order
- a score with no findings
- automatic rewrites
- bulk protected-text reproduction
- defamatory allegation generation
- one-sided evidence hiding
- objective labels for pure preferences
- a report disconnected from revision and source evidence

---

# 16. Final doctrine

```text
Review the evidence, not the target’s reputation.
Separate fact, interpretation, and preference.
Treat differences as questions before errors.
Attach sources to every professional claim.
Show counterarguments before confidence.
Use ITS to teach the missed principle.
Use PfHU to present the finding effectively.
Return self-review findings to revision.
Return critical findings to a human-authored argument.
Never clone a living critic.
Never replace human judgment.
```

SceneSmith Review Intelligence succeeds when a writer can improve their own work and a reviewer can build a fair, evidence-backed critique without losing context, uncertainty, authorship, or intellectual honesty.
